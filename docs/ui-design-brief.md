你是一名资深产品设计师 + 前端。请为我的桌面应用设计界面，产出**一个自包含的 HTML 文件（内联 CSS，可直接在浏览器打开预览）**，高质量、精致、有设计感，避免廉价的“AI 生成感”。

## 产品是什么
一个叫「**AI 辩论台**」的桌面应用（Electron）：用户输入一个**话题**，选两个 AI 当**正方/反方**，应用让这两个 AI 自动一来一回**辩论若干回合**，最后呈现一场干净的辩论记录。

## 关键技术现实（影响布局，务必体现）
- 这是**桌面 App**，不是网页。它**内嵌两个真实的 AI 网页**（如 Kimi、通义千问的网页版）在 `<webview>` 里，靠在网页里自动输入/读取来工作——所以界面里要有“**两个真实网页的实况区**”，但它是**次要的**（像后台引擎）。
- 用户主要看的是**“辩论记录”**——把双方发言按回合汇总成的干净对话流，这是**主视图**。
- 不调用任何 API；用户用自己的网页账号，所以有“**登录状态**”的概念。

## 两个页面（两步流程）

### 页面一 · 准备 / 登录
- 顶部：一个**话题输入框** + 一个醒目的「**开始辩论**」按钮。
- 一排选择器：**正方 AI**（带“支持”标签）、**反方 AI**（带“反对”标签）、**回合数**（默认 5）。每个 AI 选择器上要显示**登录状态**：未登录（红）/ 检测中（黄）/ 已登录（绿）。
- 一个**登录区**：放两个内嵌网页（webview）的框，未登录时框里显示该网站的登录页（用户在里面登录）。
- 门控：两个 AI 都“已登录”时，「开始辩论」才点亮（未就绪时置灰 + 一行提示）。

### 页面二 · 辩论
- 顶部：辩题 + 回合进度（如“第 2 / 5 回合”）+ 「复制全文」「停止」按钮。
- **左侧主区（最重要）= 辩论记录**：双方发言按回合排列，正方/反方用不同颜色区分的对话气泡，支持“正在输入”的流式动效；底部“— 辩论结束 · 共 N 回合 —”。
- **右侧次区 = 网页版实况**：两个缩小的内嵌网页框，标注“后台引擎/已登录”，带一个轻微的“运行中”指示。

## 视觉风格
- **Keynote / Apple 发布会级的精致感**：浅色背景、大量留白、考究的字体层级、柔和的大圆角、细腻的投影、克制而高级的渐变或玻璃拟态。
- 正方 / 反方用一套和谐的双色（例如 紫 vs 蓝，或 蓝 vs 暖橙），不要刺眼。
- 中文界面。桌面尺寸（约 1200–1400px 宽）。
- 拒绝：生硬的纯黑边框、廉价霓虹色、千篇一律的 Bootstrap 感。
